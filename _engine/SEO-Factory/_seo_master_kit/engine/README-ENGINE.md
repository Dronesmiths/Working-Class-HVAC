# SEO Factory Worker Engine

A standalone, modular SEO automation engine designed to be dropped into any static site project. It automatically synchronizes local SEO data with Google Sheets, fetches live metrics from Google Search Console, and generates missing content from high-converting templates.

## 📂 Structure
- `config.json`: Central configuration (Sheet IDs, Domains, Paths).
- `service-account.json`: Your Google Cloud Service Account key (keep this secret).
- `core/seo_factory.py`: The main engine logic (modular and site-agnostic).
- `core/registries/`: JSON data files for content clusters, market intelligence, and subpage plans.
- `core/templates/`: HTML templates for generating new pages (`blog.html`, `service.html`, `location.html`).

## 🚀 Setup for New Projects
1. **Drop Folder**: Copy the `seo-worker-engine/` directory into your new project.
2. **Configure**: Update `config.json` with your specific project details:
   - `SPREADSHEET_ID`: Your Google Sheet ID.
   - `GSC_SITE_URL`: Your verified GSC property (e.g., `sc-domain:example.com`).
   - `DOMAIN`: Your live website URL.
   - `WEBSITE_DIR`: Path to your website files relative to this engine.
3. **Google Keys**: Place your `service-account.json` in the engine root. Ensure the service account has 'Editor' access to the Google Sheet and 'Full' access to the GSC property.
4. **Data Registries**: Customize the JSON files in `core/registries/` to match your new project's niche.
5. **Templates**: Update the HTML templates in `core/templates/` to match your site's design.

## 🛠️ Usage
Run the engine from the terminal:
```bash
python3 core/seo_factory.py
```

### Advanced Modes
- **Autopilot (Batch generation)**:
  ```bash
  python3 core/seo_factory.py autopilot
  ```
- **Live Sync (GSC Pull)**:
  ```bash
  python3 core/seo_factory.py live
  ```
